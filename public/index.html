dsa
Usage Output Safety Install

A small CLI for finding what Docker left on disk.

Run one command on a Docker host and get a sorted report of images, volumes, logs, cache, and cleanup commands with safety notes.

$ brew install dsa
$ dsa scan --path /var/lib/docker
Install CLI Read usage

Designed to be boring in scripts.

Stable commands, stable fields, no daemon required.

dsa scan

Print a host summary with totals by Docker object type and reclaimable space.

dsa plan

Generate ordered cleanup actions with risk labels and estimated savings.

dsa json

Emit machine-readable output for CI jobs, cron reports, and platform dashboards.

dsa apply

Run approved cleanup actions while protecting running containers and attached volumes.

Readable output, stable fields.

The default report is plain text. JSON is available for automation.

Cleanup with guardrails.

The tool separates what is safe, risky, and manual-review only.

safe

Unused image tags, old build cache, and logs above your rotation threshold.

review

Detached volumes, images with recent tags, and containers with unknown labels.

manual

Database volumes, bind mounts, and anything attached to a running container.
